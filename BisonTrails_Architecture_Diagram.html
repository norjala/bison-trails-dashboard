Flow — Multi-Role Node Architecture

Deep Dive

Unlike traditional blockchains where every node does everything, Flow separates work across 5 specialized roles. Bison Trails was the only provider running all 5 node types and one of only 3 Execution Node operators.

📥
Collection
Collects transactions into clusters. Bandwidth-optimized.
🗳
Consensus
HotStuff BFT. Proposes & finalizes blocks.
⚡
Execution
Most resource-intensive. Runs transactions, maintains state.
✓
Verification
Parallel correctness checks on Execution output.
🌐
Access
Public API gateway. Routes queries & transactions.
Staking: Divide equally among 4 types (Access nodes don't stake) Execution: Only 3 operators — Bison Trails was one

Security Risk Management in the Staking Loop

Slashing Prevention

Double-signing protection locks private keys during failover. Backup can't pull a key until primary releases it. Prioritizes slash avoidance over minimizing downtime.

Downtime Mitigation

Auto-healing recovers failed nodes. Multi-cloud distribution means a single cloud outage doesn't offline the cluster. Sentry node layer absorbs DDoS.

Custody Guarantee

Staker retains full token custody. Bison Trails runs the infrastructure but never holds or can move staked assets. Keys are managed but never exposed to operators.
